# User Education — Review Checklist
*To be completed when core app features are substantially built out.*

---

## The Three Layers

Roux communicates with users through three layers working together. Before launch, every screen and feature must be reviewed against all three.

### Layer 1 — Contextual Empty States

Every screen that can be empty must explain itself warmly in one sentence. Not "Nothing here yet." — but a line that tells the user what this place is for and makes them want to fill it. Should feel like an invitation, not a dead end.

### Layer 2 — First-Use Tooltips

Non-obvious features get one tooltip the very first time a user encounters them. Appears once, never repeats. Always dismissible. Never a parade — one feature, one moment, one sentence.

### Layer 3 — Sage Education Nudges

Sage notices what users are and aren't using and introduces features at the right moment in a warm, personal tone. Uses first name. Feels like a friend noticing something, not a system issuing a notification. Examples:

- *"Hey Lauren — you've planned 10 meals. Did you know you can save your favorites as templates?"*
- *"Lauren, you haven't set up any Traditions yet. Want to add Taco Tuesday?"*
- *"Your 4th of July Cookout is coming up. Here's what you made last year."*

## Guiding Principles

- **Lauren skips tutorials.** The app must teach through doing, not reading.
- **Empty states are the first impression of every feature** — they set the tone.
- **Sage nudges should never feel like system notifications.** Always warm, always personal, always optional.
- **No feature should require documentation to understand.** If it does, the UX needs more work first.
